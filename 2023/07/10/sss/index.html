关于解构关键字…的使用

什么是解构语法

解构语法destructing,指的是用一种语法去拆解并且用变量接收一个数组或者一个对象
“解构赋值”（destructuring assignment）是一种语法，允许你使用一系列变量来接收数组或对象中的值。
它的目的是将数组或对象中的元素解构（拆解）并分配给对应的变量。

有一个不是很准确,但是描述的很形象的说法:解构语法就是把数组/对象给暴露出来

数组的解构

这样子,abcd四个变量就可以把数组内的前四个变量给暴露出来

像这样就可以直接访问数组中的元素,而不是通过数组来进行访问了

  • 如果想要获取第一个和第三个元素,中间留下一个空格即可
    let [m, ,n]=restaurant.categories;

  • 并且,返回多个变量的时候,可以使用数组返回,解构语法进行接收,可以完成接收多个变量
    function testDeStructing(){
    return [1,2,3]
    }
    //其实就算一个集体接收数组
    const [i,j,k]=testDeStructing()

如果解构的时候,如果解构的接收变量过多,则没能对应数组的数值,则会分配默认的undefined
这个时候可以使用”=”,来给变量分配默认数值

  • 另外,对于嵌套的数组,则接收形式可以是这样的

    const [[i,j],k]=[[1,2],3]仍然可以用解构语法把数组完全拆解

关于对象的解构语法

对象的解构和数组的解构有一点不同,对象本身的内部存储是没有顺序的,所以要求我们解构的时候必须指明同样的对象名字

console.log(open) 这样就能访问到嵌套的内容
默认数值的话使用’:’进行赋值
函数解构和数组解构是一样的,可以理解为另一种集体接收
本质上就是用一个内部属性完全暴露出来的数组/对象,来接收

在其他方面,这种手动解构的方式和数组大同小异,甚至用法都是一样的,但是在现代,我们似乎更偏向于使用全新的标准,也就是

在上面的解构语法中,我们可以察觉到,数组/对象这种抽象的数据结构,是被允许暴露在外的并且被调用的

但是手动解构这种东西过于臃肿了点,谁用这玩意啊…..

rest syntax

还是那句话,解构语法可以放在原本使用逗号隔开的地方

为了进一步省事,又有一种名为rest的语法

rest语法同样可以用在对象中,可以把剩余的属性打包成一个新的对象

关于位置的影响

总结:当解构运算符出现在等号左侧的时候
是进行一种打包操作,将原本按照逗号分割的物体,变成数组/对象
当解构运算符出现在右边的时候,是进行一种拆解操作
把数组/对象变成用逗号分割的形式

  • 左侧:会打包成一个数组

先这样,参数赋值的时候,底层是 …numbers=[1,2,3]

  • 右侧:把数组对象拆解开

最后总结

最后的总结其实就是一句**:在逻辑上,可以完全替换和等价于,用逗号分隔**

打包和拆分,具体按照当时的逻辑来判断
